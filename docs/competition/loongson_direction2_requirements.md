# 龙芯赛题方向二要求摘录

来源：`26嵌赛应用赛道龙芯赛题.pdf`，第 9-10 页。

## 方向名称

选题方向二：基于 LLM 的智能检测工业仪器设计。

## 方向定位

本方向聚焦人工智能与工业仪器的融合创新。项目应基于龙芯处理器平台，将大语言模型（LLM）引入工业检测、测量、控制与执行类设备，推动传统工业仪器向智能化方向升级。

## 推荐处理器平台

方向二明确鼓励使用以下龙芯处理器平台：

- 龙芯 2K1000LA
- 龙芯 2K1500
- 龙芯 2K3000

本项目以龙芯 2K1000LA 作为主控、HMI、语音、云端通信和 LLM 协同分析载体。2K0301 后续只作为传感器采集与执行控制从属板接入，避免项目主题偏移到方向三“无人智能产品”。

## 可围绕的功能

赛题原文列举的功能方向包括：

- 工业数据采集与分析
- 操作意图理解与交互优化
- 复杂参数配置辅助
- 使用指导与处理建议生成

## 面向场景

作品应面向工业生产、检测、实验与运维等应用场景。可设计为：

- 多功能检测仪
- 综合测试仪
- 工业分析终端
- 智能仪表
- 相关工业设备系统

## 价值体现

作品需要体现 LLM 技术在以下方面的价值：

- 提升工业仪器易用性
- 提升智能化水平
- 提升应用效率
- 辅助现场人员理解风险、配置参数、获取处理建议

## 赛题示意架构

PDF 第 10 页示意图表达的链路为：

```text
语音或对话
  <-> 工业仪器或设备（2K1000LA）
  <-> MCP / WebSocket
  <-> Service 云端
  <-> LLM
```

本项目当前实现可对应为：

- 语音或对话：`voice_llm_demo/`，支持持续收音、ASR 和命令理解。
- 工业仪器或设备：龙芯 2K1000LA 上运行 HMI、SafeCloud 客户端、本地规则与语音模块。
- MCP / WebSocket：当前阶段使用 HTTP/UDP discovery 做稳定联调；后续可在接口稳定后扩展 WebSocket 或 MCP。
- Service 云端：`safecloud/` 提供 `/api/evaluate`、Dashboard、设备、遥测、报警和命令接口。
- LLM：`modules/analyzer/` 支持多模型路由、真实 API、mock 回退、JudgeModel 和 SafetyGuard。

## 当前项目防偏题约束

- 主线必须是“工业智能检测仪器”，不是生产线控制系统，也不是无人车/无人机/巡检机器人。
- 传感器与执行器服务于密闭空间环境监测、风险评估、报警、排风和现场指导。
- LLM 不直接作为唯一安全执行依据；报警、排风等关键动作必须由本地规则兜底。
- 2K0301 后续负责采集与执行，但项目展示核心仍应落在 2K1000LA 工业仪器、HMI、语音交互、云端 LLM 分析和智能处置建议。
- 真实硬件未完成时，允许使用 mock 数据，但接口应保持可替换。

## 后续任务检查清单

每次多轮开发后可回看：

- 是否仍围绕工业检测仪器，而不是泛泛的云平台或无人系统？
- 是否体现了 2K1000LA 端侧仪器形态：HMI、语音、数据采集、分析结果展示？
- 是否有 LLM 带来的实际价值：意图理解、风险解释、参数辅助、处理建议？
- 是否保留本地规则安全兜底？
- 是否使用 mock 数据时保留了真实 2K0301/传感器替换接口？
- 是否能现场演示从模拟/真实数据到 HMI、语音、云端分析的闭环？
